### How to Contribute

1. **Fork the repository**
2. **Create a feature branch** - `git checkout -b fix/bug-description`
3. **Make your changes** - Follow Go best practices
4. **Add tests** - Ensure tests pass with `go test ./...`
5. **Commit with clear messages** - `git commit -m "Fix: Description"`
6. **Push and open a PR** - We'll review within 48 hours

### Code Standards

- Follow standard Go formatting (`gofmt`)
- Add comments for exported functions
- Write tests for new functionality
- Keep PRs focused (one feature/fix per PR)
